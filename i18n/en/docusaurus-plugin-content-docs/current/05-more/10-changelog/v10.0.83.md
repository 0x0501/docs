# DesireCore v10.0.83

**Release Date**: 2026-07-12

## New Features
- **Windows GUI Automation (CUA Driver)**: The Windows client now comes with a built-in cua-driver, allowing agents to operate native applications directly (screenshot, click, type, read window state) — ready to use out of the box with no extra installation.
- **Claude Subscription Integration**: Claude Pro/Max subscriptions are now supported as a compute source, with three credential methods: local Claude Code login, in-app OAuth, and manual setup-token entry.
- **Three-tier Prompt System and Centralized Prompt Hub**: A new system with global, team, and agent-level prompt layers that are injected cumulatively, all editable and managed through a unified Prompt Hub.
- **Enhanced System Prompt Management**: Agent persona/principles now support structured form editing, plus a new system prompt preview panel (including section visualization and capability declaration toggles).
- **Thinking Depth UI Control**: A thinking tier selector (7 levels) has been added to the input area, compatible with both OpenAI and Anthropic formats as well as various vendor dialects.
- **Built-in Offline Runtimes**: The installer package includes Python 3.13 and Node.js 24 LTS, allowing offline usage on first launch; when fetching the version list fails due to network issues, it falls back to the built-in manifest.
- **Provider Preset Merge Conflict Protection**: Provider fields that you have manually modified will not be overwritten by upstream changes. You will be notified of upstream updates and can review them item by item.
- **Data Backup and Migration System**: Added a "Migrate to a new device" export wizard, import identity recognition and merging, and encrypted migration of API keys.
- **Video Generation Upgrade**: Video generation has been upgraded to the OpenAI Video API protocol with access to official cloud compute, supporting Doubao Seedance 2.0.
- **File "Open With" Dropdown**: File entries in conversations now include an "Open with" option (in-app / default app / other app / show in folder).
- **Conversation PDF Export Now Client-Side**: PDF export of conversations now renders directly on the client, supporting remote deployment scenarios.
- **Provider Configuration Version Display**: The provider page header now shows the current preset configuration version, making sync issues easier to diagnose.
- **Provider Concurrency Limiting**: You can now configure the maximum concurrent requests, maximum queue length, and queue timeout per provider.
- **Plan Original Price Strikethrough & Discount Display**: Plan cards on the payment page now support showing a strikethrough original price and discount information.
- **Agent File Revert Upgraded to Full Git Tree Snapshots**: Agent file reversion is now more complete, supporting a wider range of restore scenarios.
- **Manual Refresh for Service Catalog**: A refresh button has been added to the "Apps & Services" page for manually syncing the service catalog.
- **Automatic Context Overflow Recovery & Dynamic Output Budget**: When a model exceeds its context window, it now automatically learns the true window size and transparently retries, and large output tasks are completed in one go whenever possible.
- **Delivery Edges in the Relationship Graph**: The relationship graph now correctly displays "delivery" edges when results are successfully delivered.

## Fixes
- **Ls Directory Listing Executed Without Approval**: Listing directory contents now executes directly without requiring approval; clear authorization guidance is shown when system permissions are insufficient.
- **Duplicate Teams in Agent List**: Fixed an issue where AI could repeatedly create teams with the same name, causing multiple identical teams to appear in the list.
- **Agent Offline After Import**: Imported or cloned agents no longer display as "offline."
- **Data Correctness in Concurrent Multi-Agent Delegation**: Fixed potential data loss, file corruption, and directory conflicts that could occur during parallel delegation.
- **Accurate List Status Indicator**: Agents now correctly show as yellow (busy) rather than green (online) when using tools.
- **Wider Message Scrollbar**: The click/grabbable area of the chat area scrollbar has been increased from 4px to 10px for easier dragging.
- **Long Content Fully Visible**: Very long URLs, paths, or wide tables no longer cause horizontal scrollbars; content now wraps or scrolls internally as appropriate.
- **Short Message Action Bar No Longer Covers Timestamps**: Buttons now wrap automatically in narrow bubbles, so they no longer cover the timestamp.
- **No More Flicker When Scrolling to the Bottom of the Message List**: Fixed an issue where continuing to scroll after reaching the bottom would cause the whole window to flicker.
- **Scheduled Tasks Truly Execute**: Scheduled tasks now complete multi-step tool calls when triggered, instead of being cut off after a single reply.
- **Scheduled Task Failures No Longer Infinite Retries**: Failed tasks now correctly follow an exponential backoff strategy instead of retrying indefinitely.
- **Time Drift Fix for Long-Running Sessions**: The agent's "current time" no longer stays stuck at the moment the session started.
- **Scanned Documents No Longer Crash with Non-Vision Models**: A prompt to switch to a vision model is now shown directly, instead of entering an infinite repetitive output loop.
- **file:// Links No Longer Crash**: Clicking local file links now opens them with the system default application instead of causing errors or freezing.
- **Windows Git Paths with Spaces No Longer Fail**: Fixed all Git operation failures that occurred in paths like Program Files.
- **Service Catalog Status Updates in Real Time**: The catalog now auto-refreshes after approval or registration, and the pending approval panel no longer crashes due to dirty data.
- **Clearing Data No Longer Fails on Windows**: Fixed clearing failures caused by read-only git files and reserved device name files.
- **Model Routing Conflict with Same-Named Models Fixed**: Official cloud and locally configured models with the same name are no longer incorrectly routed.
- **Thinking Depth Budget Collapse Fixed**: The 5 tiers now use their correct budget values, with the deepest tier reaching 32K+.
- **Cursor Auto-Scrolls into View After Line Break in Input Box**: After Shift+Enter, accepting predictions, and similar actions, the cursor always remains visible.
- **Delegation Card Correctly Displays Target Name**: Fan-out delegations no longer display as "unknown."
- **Concurrent Message Reply Matching Fixed**: When multiple messages are sent concurrently from the same sender to the same target, replies are no longer mismatched.
- **Queued Queries Can Be Cancelled**: Clicking the stop button while a query is queued now immediately dequeues it rather than allowing it to be scheduled and executed.
- **Provider Concurrency Limiting Refined**: The rate limiter uniformly identifies providers by ID, provides status feedback to users while requests are queued, and supports diagnostic views.
- **Partial DeepSeek-V4 400 Errors Self-Healing**: Fixed two types of 400 errors on desirecore-cloud related to max_tokens range and thinking backpass that could not self-heal.
- **Windows Backslash Loss Fixed in Paths**: Paths in system prompts are now wrapped with inline code formatting, preventing backslashes from being lost during display and copying.
- **Pointer Skills Restored in the Marketplace**: 20 pointer-type skills that were incorrectly filtered out have been restored.
- **macOS Notarization Fix**: Fixed an issue where macOS builds were rejected by notarization due to the built-in Python signing.

## Improvements
- **Out-of-the-Box Runtime Environment**: Toolchain directories have been unified under `~/.desirecore/runtime`; existing users are automatically migrated to the new directory structure once during the upgrade.
- **Standalone Deployment Mode**: agent-service now supports running as a standalone server independent of the desktop client (single tenant, remote secure access, one customer per container).
- **Claude SDK Backend Enhancements**: Added request queuing and rate limiting, cold start time observance, a cancel button during login, session resumption, and WebSearch support.
- **Comprehensive Multi-Concurrency Governance**: Process locks prevent multiple instances, a global concurrency slot queue (cancellable, priority-aware), provider-level semaphores, and fully atomic write paths.
- **Agent Capability Boundary Enforcement**: The Claude Agent SDK is now responsible only for LLM requests, while tools, skills, permissions, and file execution are fully managed by the DesireCore governance chain.
- **System Prompt Streamlining**: Removed redundant self-identification and deprecated capability declarations, saving token consumption.
- **Task Board Probing Silenced**: Switching sessions no longer prints 404 error logs to the browser console.
- **Subtask Timeout Relaxed**: The wait timeout for delegate / spawn_agent / send_message has been extended from 5 minutes to 30 minutes, and the subtask is no longer forcibly aborted on timeout.
- **Data Import/Export Reliability Enhanced**: Fixed a zip slip vulnerability and a service outage after import failure; added manifest and identity boundary detection.
- **credentialSource Anti-Poison Pill**: Changed to an open string type so that unknown values no longer crash the configuration; corrupted configurations at startup are automatically downgraded and rebuilt.
- **Standalone Authentication Hardened**: Unified the authentication entry point for requests and added media signed URLs to ensure access to images and videos in remote scenarios.
- **Development and Stability Improvements**: Fixed various issues including a blank screen on first load in worktree development mode, session recovery after sleep/hibernate, and stream request interruptions.
