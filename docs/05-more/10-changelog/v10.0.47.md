# DesireCore v10.0.47

**发布日期**: 2026-05-06

![v10.0.47 更新概览](/img/changelog/v10.0.47.png)

## 新功能
- **浏览器自动化能力（web-access v2.0）**：智能体现在可以在激活 web-access 技能后直接操作浏览器标签页、页面跳转、点击、滚动、截图、上传文件，并可结合本地书签与历史记录，更高效地处理登录态网站任务。
- **站点经验积累（SitePattern）**：智能体现在可以读取和写入站点操作经验，逐步沉淀对小红书、B站、微博、知乎、飞书等站点的可复用操作知识。
- **智能体级全局技能排除**：你现在可以只对当前智能体隐藏某个全局技能，而不会影响其他智能体继续使用该技能。

## 修复
- 更新下载完成后，更新窗口现在会正确显示“立即重启”，不再出现与当前状态不符的按钮。
- 应用更新在下载失败时现在会自动切换备用镜像并重试，同时在界面上明确提示重试或失败原因，减少更新卡住的情况。
- 修复了从某个智能体中移除全局技能时误影响所有智能体的问题，避免共享技能被错误删除。
- 修复了部分技能在后续对话轮次中已激活却无法继续调用对应工具的问题，使连续任务中的技能能力保持可用。
- 修复了此前心跳系统改动引发的大量界面文案缺失与页面报错问题，聊天与编辑相关页面恢复稳定可用。
- 修复了表单相关依赖版本不一致导致的启动/构建异常，避免部分配置界面无法正常打开。

## 改进
- 激活技能前，重型工具默认不会暴露给模型，日常对话更轻量，也能减少不必要的上下文消耗。
- 浏览器访问相关能力补充了更严格的安全限制，包括网址协议、文件名、站点域名与敏感内容写入控制，使用过程更安全。
- 本地书签与历史记录检索在部分环境缺少依赖时会自动降级处理，尽量保证功能可用而不是直接失败。
- 更新下载过程新增“正在准备下载”和“正在切换备用源重试”等更清晰的进度反馈，状态变化更容易理解。
