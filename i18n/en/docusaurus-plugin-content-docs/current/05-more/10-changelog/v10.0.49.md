# DesireCore v10.0.49

**Release Date**: 2026-05-08

![v10.0.49 更新概览](/img/changelog/v10.0.49-en.png)

## New Features
- **Enhanced support for DashScope encoding options**: Added the DashScope Token option and improved related model and provider presets, giving you more ready-to-use DashScope choices when configuring encoding and compute plans.

## Fixes
- When an agent runs "Check for Updates," if the remote repository is temporarily unavailable or there is a network issue, it will now clearly indicate that fetching the remote version failed instead of incorrectly reporting "Already up to date."
- When an agent’s automatic update encounters a Git conflict, it will now display the conflicting files directly and keep a conflict indicator visible at the top of the chat, making it easier for you to locate and resolve the issue.
- After creating a new agent, Git repository initialization is now more stable, so immediately running Git initialization, checking status, or viewing history is less likely to fail due to concurrency conflicts.
- During automatic agent updates, pulls will now prioritize the branch configured in the repository, reducing update issues caused by branch mismatches.
- During automatic updates, different failure causes—such as no remote repository configured, invalid configuration, or the current directory not being a Git repository—will now produce more accurate prompts instead of a single generic failure message.

## Improvements
- When manually checking for desktop app updates, the app will now fetch the version list for your current operating system, so you can see new releases for your platform sooner.
- When proactively checking for agent updates, the system will now bypass stale caches and fetch remote information again, allowing newly released versions to be detected more promptly.
