# 翻译助手

## 真实工作流

这个场景不是单句翻译，而是面向真实读者的交付翻译，重点在术语、语气和上下文一致性。

| 维度 | 真实情况 |
|------|----------|
| 触发点 | 给海外客户发 API 文档、回复外文邮件、准备产品本地化内容 |
| 现有材料 | 原文、目标语言、读者身份、产品术语表、历史翻译风格 |
| 卡点 | 术语不统一，直译生硬，代码、接口名、法律或商务措辞容易出错 |
| DesireCore 介入 | 翻译助手按术语表生成译文，并列出关键术语、风格选择和歧义句 |
| 验收结果 | 用户拿到可审校译文，重点检查专业术语、客户敏感表达和上下文一致性 |

## 推荐智能体

**翻译助手** -- 支持中英日韩法德西俄等主流语言互译，适合商务文档、技术资料、产品本地化和跨语言邮件沟通。

## 前期教学：让智能体掌握你的术语和风格

> 长期翻译项目最怕术语不一致。同一个产品名、功能名或行业词，在不同文档里应保持同一译法。

### 教规则

```
你: 我把我们产品的官方术语表传给你，以后翻译都要严格按这个来。
    [上传 产品术语表_中英对照.xlsx]
    另外还有几条风格要求：
    1. 统一使用美式英语拼写，比如 organize 不用 organise（强制）
    2. 技术文档：简洁直接，不要过度礼貌的措辞（偏好）
    3. 营销文档：语气积极，多用短句增加节奏感（偏好）

翻译助手: 📝 学到了新内容
          来源：产品术语表_中英对照.xlsx（共 247 个术语对）+ 本次说明
          类型：规则（翻译标准）
          层级：强制 × 1 项（含术语表），偏好 × 2 项

          我的理解：
          翻译时严格使用术语表中的对照翻译，不自行发挥。拼写统一
          美式英语。技术文档走简洁风，营销文档走活泼风。

          [✓ 正确]  [✎ 修改]  [✗ 撤销]

你: [✓ 正确] 对了，术语表如果后面有更新我会再传给你。
```

教学完成后，翻译助手已掌握你的术语表和风格要求。之后提供文档时，它会按这套术语和风格生成译文。

## 完整对话示例

```
你: 帮我把这份 API 文档翻译成英文，注意技术术语要准确。

    "用户认证
    所有 API 请求必须在请求头中包含认证令牌。
    请求频率限制：
    - 免费版：每小时 100 次请求
    - 专业版：每小时 10,000 次请求"

翻译助手: 识别到文本类型：技术文档
         翻译结果：

         Authentication
         All API requests must include an authentication token
         in the request header.

         Rate Limiting:
         - Free tier: 100 requests per hour
         - Pro tier: 10,000 requests per hour

         术语对照：
         - 认证令牌 → authentication token
         - 请求头 → request header
         - 请求频率限制 → rate limiting

         代码片段已保持原样不翻译。

你: 再帮我翻译一封客户邮件，从中文翻成英文，
    要正式一些，商务风格。

    "感谢贵方对我司产品的关注。关于定制化需求，
    我们的技术团队已完成评估，预计开发周期为 2 周。"

翻译助手: 识别到文本类型：商务邮件（正式）

         Thank you for your interest in our products.
         Regarding your customization requirements,
         our technical team has completed the evaluation.
         The estimated development timeline is two weeks.

         翻译要点：
         - "贵方"在英文中不需要对等表达，直接用"your"
         - "我司"翻译为"our"即可，避免过度正式
         - 数字"2 周"使用"two weeks"拼写形式，更符合正式邮件规范

        📋 回执摘要
        引用的术语表：产品术语表 v2.1（共匹配 23 个术语）
        引用的规则：美式英语拼写 · 技术文档简洁风格
        均来自你的教学（上月）✓
```

## 关键步骤

1. **定义读者和用途** -- 同一段中文，给开发者、客户、法务或市场看的译法会不同
2. **准备本地化包** -- 提供术语表、产品名、变量/占位符规则、风格指南和历史译文
3. **按上下文翻译** -- 技术文档保留代码和接口名，营销文案保留语气，UI 文案关注长度和按钮语境
4. **术语与格式 QA** -- 检查术语一致性、数字单位、链接、变量、标点和不可翻译内容
5. **沉淀翻译记忆** -- 把确认过的术语和表达沉淀下来，后续批量翻译保持一致

## 最终成果

一份可审校的译文，通常会附带术语对照、风格说明、不可翻译项和原文中容易产生歧义的句子。

:::tip 最佳实践
- 翻译前告诉智能体文档用途和读者对象（如"翻译给美国开发者看的 API 文档"），译文风格会更稳定
- 如果有公司内部的术语表，可以一开始就提供，智能体会严格遵循
- 对于长文档，建议分批翻译并在每批之间确认术语一致性
- 法律文件、合同等关键文档的翻译结果建议由法律专业人士做最终审核
- 批量翻译前先确认统一术语表和风格要求，再把多份文档交给翻译助手处理
- 产品界面和帮助中心翻译时，要检查变量、按钮长度、链接和截图上下文，不能只看句子本身
:::
