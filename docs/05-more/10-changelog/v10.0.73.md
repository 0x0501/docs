# DesireCore v10.0.73

**发布日期**: 2026-06-05

![v10.0.73 更新概览](/img/changelog/v10.0.73.png)

## 新功能
- **本地离线 draw.io 绘图**：现在你可以在对话中生成图表，并在 Agent 资源或 Markdown 中编辑 draw.io 图表，即使断网也能使用完整的专业绘图能力。
- **克隆智能体**：聊天界面新增“克隆智能体”，可快速复制现有智能体并自动切换到副本，方便基于已有配置继续修改。
- **更安全的外部服务授权**：智能体访问 HTTP、MCP、脚本或 Bash 工具时，可以在不暴露明文凭据给智能体和模型的前提下完成授权访问。
- **PDF 智能阅读分流**：读取 PDF 时会优先提取文字页内容、仅在图表或扫描页时调用视觉能力，因此阅读更快、支持更多页数且更不容易占满上下文。
- **运行环境注入与刷新反馈**：智能体现在能感知更完整的系统与运行时环境信息，同时在“运行环境”页手动刷新后会明确提示是否刷新成功。

## 修复
- 修复了从桌面图标或启动器打开应用时，智能体可能找不到 Homebrew、nvm、conda 等本机工具的问题。
- 修复了部分 zsh 环境下 PATH 解析不完整的问题，使写在 `.zshrc` 中的工具路径也能被正确识别。
- 修复了某些情况下 `compute.json` 配置损坏后模型列表清空的问题，并在配置异常时自动尝试恢复。
- 修复了 Kimi 在 thinking 模式下进行多轮工具调用时可能报错中断的问题。
- 修复了 SuperDoc 内联 diff 审阅中接受变更会撤销、定位错位、纯删除无法拒绝等问题。
- 修复了“资源管理/切换模型”中“使用默认配置”文案被挤成竖排显示的问题。

## 改进
- **切换模型提示更清晰**：在资源管理面板中，切换模型时会明确提示这不会影响当前会话和记忆，减少误操作顾虑。
- **Office 技能离线依赖预装**：文档类技能所需依赖会随应用预装并跨平台复用，首次使用更快，也更适合离线场景。
- 浏览器访问相关能力在浏览器被关闭或失联后可自动恢复，不再更容易卡在指定浏览器状态。
- 内置技能市场同步了新版本内容，已升级用户可正常收到部分全局技能的更新。
