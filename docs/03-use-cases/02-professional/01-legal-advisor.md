# 法律顾问

## 真实工作流

法律场景的真实性在于“预审”，不是让智能体替代法务下结论。它适合在正式法务审查前，把合同问题先整理出来。

| 维度 | 真实情况 |
|------|----------|
| 触发点 | 采购合同、外包合同、保密协议、客户协议进入审批前 |
| 现有材料 | 合同文本、公司审查标准、历史审查报告、业务背景和金额信息 |
| 卡点 | 条款多，业务负责人不确定哪些地方要问法务，法务沟通成本高 |
| DesireCore 介入 | 法律顾问按公司标准标出风险、建议改法、需补充背景和需法务确认的问题 |
| 验收结果 | 业务负责人拿到预审清单，再和法务集中讨论关键条款 |

## 推荐智能体

**法律顾问** -- 适合合同条款检查、法规资料检索、合规要点整理和风险提示。

## 前期教学：让智能体掌握你的审查标准

> 建议先把公司的合同审查标准、红线条款和报告格式教给智能体。之后它会按同一套规则做预审。

### 教规则

```
你: 教你我们公司审查采购合同的标准：
    1. 付款条件：必须是"验收合格后 30 天内付款"，预付款不超过 30%（强制）
    2. 违约金：不超过合同总金额的 20%（强制）
    3. 合同期限：不超过 3 年（强制）
    4. 知识产权：定制开发成果归甲方所有（强制）
    5. 审查报告格式：先摘要表格，再风险点列表，最后综合评级（偏好）

法律顾问: 📝 学到了新内容
          类型：规则（采购合同审查标准）
          层级：强制 × 4 项，偏好 × 1 项

          我的理解：
          审查采购合同时，强制检查付款条件（≤30% 预付 + 验收后 30 天
          内付款）、违约金（≤20%）、期限（≤3 年）、知识产权（定制成果
          归甲方）。报告按"摘要表格→风险列表→综合评级"格式输出。

          [✓ 正确]  [✎ 修改]  [✗ 撤销]

你: [✓ 正确]
```


教学完成后，法律顾问已掌握你公司的合同审查标准。之后提供合同时，它会按这套标准做预审。

## 完整对话示例

```
你: 请帮我审查这份供应商合同，明天就要签约了。
    [粘贴合同内容]

法律顾问: 我来做一轮合同预审。正在分析条款...

        审查结果摘要

        合同主体：清晰明确
        付款条款：存在风险
        违约责任：基本合理
        争议解决：建议修改

        重点关注事项

        1. 付款条款 -- 中风险
           问题：预付款比例过高（50%），且未约定进度节点
           建议：改为 30% 预付 + 40% 进度款 + 30% 验收款
           依据：《民法典》合同编相关规定

        2. 争议解决 -- 低风险
           问题：约定仲裁但未明确仲裁机构
           建议：明确约定具体仲裁委员会名称

        3. 知识产权 -- 需关注
           问题：未明确定制开发成果的知识产权归属
           建议：补充约定"定制成果归甲方所有"

        总体评估
        合同基本可用，建议修改上述 3 处后再签署。
        需要我生成修改建议稿吗？

你: 帮我生成修改建议稿，重点改付款条款。

法律顾问: 已生成修改建议稿。
        付款条款修改建议：
        原文："甲方应在合同签订后 5 个工作日内支付合同
              总价的 50% 作为预付款..."
        修改为："甲方按以下节点分期支付：
              (1) 合同签订后 5 个工作日内支付 30% 作为预付款；
              (2) 乙方完成系统部署并通过中期验收后 5 个工作日内
                  支付 40%；
              (3) 项目终验通过后 10 个工作日内支付剩余 30%。"
        ...

        📋 回执摘要
        引用的规则：付款条件标准 · 违约金上限 · 知识产权归属
        均来自你的教学（3 天前）✓
```

## 关键步骤

1. **确认合同背景** -- 先说明合同类型、金额、交易对象、业务目的和最关心的条款
2. **套用公司红线** -- 对付款、违约、期限、知识产权、保密、争议解决等条款按公司标准检查
3. **生成风险矩阵** -- 每个问题都标注条款位置、风险等级、业务影响、建议改法和是否必须法务确认
4. **输出谈判文本** -- 对高风险条款生成可直接拿去沟通的替换文字
5. **形成法务沟通清单** -- 把 AI 无法判断、需要业务或法务补背景的问题单独列出

## 最终成果

一份合同预审报告，包含风险点、风险等级、建议修改文本和需要法务确认的问题清单。

:::warning 免责声明
法律顾问智能体提供的是辅助分析，不构成正式法律意见。涉及大额合同、诉讼、股权交易、劳动争议等事项时，应由执业律师或公司法务做最终审查。
:::

:::tip 最佳实践
- 上传合同时一并说明你最关心的条款（如付款方式、违约责任），智能体会重点分析这些部分
- 如果涉及特定行业（如建筑工程、IT 外包），告诉智能体行业背景和适用地区，便于它按更接近实际的规则整理风险
- 可以让法律顾问和财务助手协作，一个审条款一个算成本，全面评估合同
- 批量审查合同时，建议先统一审查口径，再让法律顾问按同一标准输出风险清单
- 合同预审最重要的不是“指出很多问题”，而是区分哪些必须改、哪些可谈判、哪些只需业务知情
:::
