# DesireCore v10.0.83

**Release Date**: 2026-07-12

## New Features
- **Claude Subscription Integration**: Support using Claude Pro/Max subscriptions as a compute source, connectable via local Claude Code login, in-app OAuth, or manual token entry. All model calls are fulfilled through the built-in Agent SDK, fully preserving approval and audit trails.
- **Thinking Depth Control**: A thinking depth level selector (Auto/Off/Light ~ Deepest) has been added to the chat input area. Supports per-session switching and can be set as a global default, compatible with both OpenAI and Anthropic formats.
- **Three-Tier Prompt System**: Introduces global layer (personal general rules), team layer (shared team rules), and Agent layer prompts, injected in three layers. The resource manager now includes a "Prompt Center" for unified management of prompts at each layer, supporting structured editing and preview.
- **Migration to a New Device**: A "Migrate to a new device" guided wizard has been added to the Settings page. It can export full data (including optional encrypted API keys) and supports identity replacement or merging into the current identity when importing on a new device.
- **API Key Migration**: The migration export wizard now offers an option to encrypt and export API keys, protected by a password with AES-256-GCM encryption. Import supports per-Provider replacement/merge conflict resolution.
- **Windows GUI Automation**: The Windows installer bundles the cua-driver runtime, enabling agents to operate Windows desktop applications out of the box (screenshots, clicks, input, reading window state) without manual installation.
- **Built-in Offline Runtimes**: The installer includes Python 3.13.9 and Node.js 24.18 LTS offline runtimes, silently imported in the background on first launch. Falls back to offline installation automatically when network fails, completely eliminating environment anomalies caused by network issues.
- **File "Open With" Menu**: Files generated or referenced in chats now have an "Open With ▾" dropdown menu, allowing in-app opening, system default app, other apps, or locating in folder. Unifies file opening logic across three contexts.
- **AgentFS Rollback Upgraded to Git Full Tree Snapshot**: The rollback feature now uses native git mechanisms for file restoration, supporting precise rollback across multiple rounds. Git binary is bundled with the installer to ensure deterministic behavior.
- **Compute Preset Update Review**: When config-center pushes new compute configuration presets, the client now displays a visual review panel — showing field-level diffs per Provider. Users can choose "Adopt upstream" or "Keep mine" item by item, with a notification badge reminder.
- **Single-Tenant Standalone Deployment**: Agent Service now supports deployment on a server independent of Electron, with a Docker one-customer-one-container delivery solution. Supports secure remote access (instance token authentication + loopback mesh + configurable CORS).
- **Context Overflow Transparent Self-Healing**: Automatically learns the real context window size from model-returned "context length exceeded" errors. Subsequent conversations trim directly by the actual value; on overflow, automatically reduces output or re-trims input with the true window, no user intervention needed.
- **Video Generation Integration with Doubao Seedance 2.0**: The video generation tool now supports the Doubao Seedance 2.0 model, automatically adapting to channel format differences.
- **Agent Compute Configuration Capability**: A new official "Configure Compute" skill enables Agents to configure API keys for users (write-only, no read) and models. On the frontend Provider detail page, key display is now masked and viewable only on demand.
- **Queue Concurrency Control**: When multiple Agents send messages simultaneously, requests exceeding the concurrency limit are automatically queued, displaying "Queued (position N)". Users can cancel queued requests at any time.

## Fixes
- **Delegated Multi-Agent Concurrency Data Safety**: Fixed potential data loss, directory collisions, counter field drift, file half-write corruption, etc., when delegating in parallel, ensuring correct behavior at any concurrency level.
- **Concurrent Slot Permanent Leak After "Stop"**: Fixed global concurrency slots not being released after manual generation stop, causing all Agent top-level queries to hang permanently after 3 stops. Also patched the same root leak on system sleep and session reset paths.
- **Scheduled Tasks Not Executing / Failed Hard Retry**: Fixed scheduled tasks expiring and being truncated after a single reply (too tight turn limit), cyclic tasks retrying infinitely on failure without backoff, and time display misalignment after long tasks.
- **Marketplace Pointer Skills Invisible**: Fixed all pointer-type skills in the marketplace being unbrowseable and unsearchable. Also fixed upstream skills without version numbers having their updates silently skipped forever.
- **Model Name Collision Cross-Routing**: Fixed cloud models sharing the same name as local Provider models (e.g., deepseek-v4-pro) being incorrectly routed to the local Provider, covering all LLM call paths including conversations, delegation, and heartbeats.
- **Imported Team Agents Shown Offline**: Fixed imported or cloned agents appearing offline in the list, only returning online after a restart.
- **Scanned Document with Non-Vision Model Degraded Looping**: Fixed non-vision models encountering pure scanned PDFs entering infinite repeat output — now directly prompts to switch to a vision-capable model and ends the turn.
- **List Status Indicator Inaccurate**: Fixed the chat list status light still showing green instead of yellow when an Agent is using a tool (e.g., Read).
- **Delegation Multi-Target Validation Gap**: Fixed fan-out delegation to non-existent targets silently creating empty shell sessions and displaying "Completed". Also fixed delegation cards showing "unknown" for multiple targets.
- **AgentFS Hot Reload Invalid**: Fixed directly editing agent.json not triggering a hot update (root cause: file watcher incorrectly treated the entire root directory as hidden and ignored it).
- **Default Model Path ProviderId Missing/Mismatch**: Fixed missing providerId in legacy data causing the default model to permanently drift and misroute to the wrong Provider.
- **Systematic Fix for credentialSource Poison Pill**: Changed the credential source field from a hard enum to an open string, preventing future new credential source types from causing complete compute functionality paralysis on older clients.
- **glm Pathological Output Killing Session**: Fixed certain models outputting tool parameters as arrays, causing a gateway error "list object has no attribute items" that made the entire session unrecoverable.
- **Self-Healing for Two Types of 400 Errors on Anthropic-Compatible Gateways**: Fixed 400 errors caused by max_tokens range limits and missing thinking block return on third-party Anthropic-compatible gateways for models like deepseek-v4.
- **Relationship Graph Missing "Deliver" Edge**: Fixed Agent delivery result events being silently dropped, preventing the relationship graph from ever drawing a "deliver" edge.
- **send_message Async Receipt Mismatch**: Fixed async receipts from concurrent send_message calls potentially being consumed by the wrong synchronous waiter.
- **Windows Path with Spaces Git Crash**: Fixed all git operations failing on Windows due to git paths containing spaces.
- **Windows "Clear Data" Failure**: Fixed read-only git files and reserved device name files causing an EPERM failure during the clear data operation.
- **Clicking file:// Link Crash**: Fixed clicking local file links in Agent responses causing a macOS error or Windows gray screen freeze; now opened by the system default app.
- **Cursor Invisible After Line Break in Chat Input**: Fixed the cursor falling outside the visible area after Shift+Enter line breaks when the input area exceeds the height limit, making users think the line break didn't work.
- **Horizontal Scrollbar and Long Content Clipping in Message Area**: Fixed long URLs, oversized tables, etc., causing a horizontal scrollbar or content clipping in the message area; content is now always fully visible.
- **Message Action Bar Covering Timestamp**: Fixed action buttons overflowing and covering the right-side timestamp on short messages.
- **Clicking Selection Circle Unresponsive in Multi-Select Mode**: Fixed occasional unresponsiveness when clicking the left selection circle in chat multi-select mode.
- **Backslash Loss in Windows Paths Within System Prompts**: Fixed backslashes being swallowed by Markdown escaping when the Agent restated Windows paths; paths are now wrapped in inline code.
- **Whole Window Flicker When Scrolling to Bottom of Message List**: Fixed trackpad scrolling past the bottom of the message list causing the entire window to flicker.
- **Team Workspace Directory Path Error**: Fixed incorrect paths for team shared workspace directories during delegation and context injection.
- **Codex Off-Level Thinking Not Explicitly Disabled**: Fixed reasoning parameters possibly still being sent when the Codex model's thinking was set to off.
- **Topic Detail Message Count Display Abnormal**: Fixed message counts in topic details displaying values in the hundreds of millions.
- **Pending Review Window Dirty Data Render Crash**: Fixed missing fields in some service entries registered by an Agent causing the pending review page to crash.
- **Compat Gateway Thinking Parameters Missing**: Fixed no thinking parameters being sent when a thinking level was selected on gateways that do not declare thinkingFormat.
- **Marketplace Repo Save Stuck**: Fixed adding a repo source and saving showing "Saving..." indefinitely and displaying the repo twice.

## Improvements
- **Thinking Depth Budget System**: Fixed the UI showing 5 depth levels but only producing 3 budget values, making "Deep" and "Deepest" indistinguishable. Thinking levels are now persisted to agent.json and retained after restart. Existing legacy fixed budget groups are auto-identified and ignored.
- **Dynamic Output Budget and Large Output One-Shot Generation**: Output token budget is now calculated dynamically, no longer limited by a static cap or manual step-by-step continuation. Large outputs can be generated in one go.
- **Context Trimming and Token Estimation Optimization**: Trimming now uses a cascade strategy (truncate tool_result → delete oldest turns → mid-fold → break soft floor), eliminating deadlocks where trimmed context still exceeds the limit and is sent as-is. Fixed undercounting of thinking, tool names, and array contents in token estimation, improving accuracy.
- **Provider Concurrent Rate Limiting Hardening**: Provider-level concurrency semaphore adds a bounded queue, queue timeout, cancellability, hot reload config retry, and diagnostics, preventing instantaneous requests from overwhelming upstream.
- **compat-proxy Precise Routing by ProviderId**: The proxy layer adds a three-layer routing defense: by ProviderId, by inbound API key fingerprint match, and fallback by name, preventing cross-routing of models with the same name across different Providers.
- **Queued Queries Can Be Canceled**: Queries waiting in queue now support manual user stop or auto-cancellation on disconnect, preventing resource consumption for irreversible tool calls. Interactive queries are prioritized over background tasks.
- **Pipeline Assembly Stage Cancelable**: Queries in the tool assembly and context building preparation phases now immediately abort upon receiving a cancel signal, preventing long blocking followed by launching a meaningless query.
- **Provider Semaphore Queue-Aware Cancellation**: If a request is aborted while waiting for a Provider concurrency slot, it is immediately removed from the queue, no longer wasting queue time.
- **via-messaging Precise Reply Matching**: Delegation/messaging/assistance requests now include a correlationId linking field, preventing reply mismatch in concurrent requests within the same conversation.
- **Subtask Sync Wait Timeout Relaxed to 30 Minutes**: The timeout for delegation/spawn_agent/send_message subtasks has been relaxed from 5 minutes to 30 minutes. Timeouts no longer auto-abort the task; the caller decides whether to continue waiting or proactively end.
- **Ls Tool Default Direct Execution Without Approval**: Ls (list directory), as a read-only safe tool, now defaults to direct execution. Permission control is delegated to the OS layer, and platform-specific authorization guidance is returned only when permission is denied.
- **Agent Configuration Key Write-Only No-Read**: API keys in frontend responses are now masked (first 4 characters + •••). File tools (Read/Grep/Glob) precisely deny reading key files. Bash built-in blacklist blocks commands containing key file names.
- **SSRF Protection Per-Agent Granularity**: SSRF protection now supports toggling per Agent with immediate effect. DNS Rebinding protection has been added, locking DNS resolution results before fetch.
- **Standalone Auth and Remote Media Access Hardening**: Unified the Agent Service remote request authentication entry point. Native media elements now use time-limited HMAC-signed URLs with 5-minute validity to prevent long-term token exposure.
- **Three-Tier Prompt System**: Global/team/Agent three-layer overlay injection. Centralized Prompt Center supports visual editing and preview. The systemPrompt structure is displayed grouped by global/team/agent/system.
- **Compute Page Shows Preset Config Version**: The header of the compute configuration page now displays the merged config-center preset version number for easier troubleshooting.
- **Provider-Level requiredClientVersion Gating**: When the client version is lower than the Provider's requirement, that Provider is automatically filtered from the model selector and a prompt to upgrade the client is shown.
- **Compute Configuration Resilience Enhancement**: Salvage protection is now enabled for fresh installs or damaged rebuild paths. A single broken Provider no longer blocks the entire config loading or Agent Service startup.
- **Conversation PDF Export Switched to Client-Side Rendering**: PDF export no longer relies on server-side rendering; it now uses Electron printToPDF, decoupling it for Agent Service standalone deployment.
- **Standalone Deployment Data Externalization**: All per-instance data paths for Agent Service standalone deployment are now unified under DESIRECORE_HOME. TLS certificates are no longer placed in the bundle directory.
- **DESIRECORE_HOME Process Lock**: Agent Service acquires a process lock on the data root directory at startup. On multi-process conflict, it queues and waits instead of throwing an error, preventing simultaneous writes to the same data directory by multiple processes.
- **Write Path Atomicity Hardening**: All whole-repository overwrite writes are now atomic (temp file + rename), eliminating file half-write corruption from crashes. The agent.json dual lock system has been unified into a per-agent serial lock.
- **Import/Export Rules Single-Sourcing**: Duplicate import/export exclusion rules in main and agent-service have been converged into a shared module, eliminating rule drift. Legacy dead code has been removed.
- **Runtime Directory Unification**: The runtime toolchain directory is now unified to ~/.desirecore/runtime, eliminating forks across environments. Existing users are auto-migrated on first launch.
- **Message Scrollbar Wider Hit Area**: The chat message area scrollbar hit area has been expanded from 4px to 10px for easier mouse dragging.
- **Relationship Heat Formatted Display**: Relationship heat is now displayed with thousand separators and one decimal (e.g., 3,529.6), no longer rendering long floats that break the card layout.
- **Apps & Services Page Adds Manual Refresh Button**: A refresh button has been added to the title bar of the Apps & Services page for proactively syncing the service registration directory.
- **Service Directory Status Real-Time Update**: Service registration status changes are now pushed to the frontend in real time via Socket, eliminating the need for manual refresh.
- **Compute Provider Notice and Billing Info**: Official compute model displays a notice at the top: "Compute powered by partner {provider}". Per-token or per-use billing modes show pricing and billing basis.
- **Task Board Detail Probe Eliminates Console Noise**: Task board probe requests on session switch no longer generate a 404 in the console; changed to normal 200 probe semantics.
- **WorkflowRun Wait Timeout Relaxed**: The WorkflowRun tool now defaults to no tool-level timeout, with an optional parameter to precisely control wait time. Long workflows are no longer prematurely judged as timed out.
- **Delegation Prompt Path Rules**: The delegation tool prompt now includes path rule guidance: cross-Agent resource transfer must use absolute paths.
- **Chat Message List Scrollbar Flicker Prevention**: Added overscroll-contain to prevent whole-window flicker and updated the global fallback rule.
- **Multi-Select Mode Accessibility Improvements**: Selection circles have been changed to semantic buttons, with added screen reader support and keyboard accessibility.
- **Marketplace Pointer Item Upgrade Gating**: The upgrade version threshold field for pointer items is now correctly displayed, consistent with inline item behavior.
- **Plan Card Original Price Strikethrough**: Plan pricing now supports displaying the original price with a strikethrough and a discount percentage/free label.
- **System Prompt Simplification and Dedup**: Removed duplicate path listings and zombie capability statements in self-cognition prompts, saving tokens.
