# 企业服务协调

## 真实工作流

企业服务协调场景成立于跨角色、跨材料、跨时间线的任务。真实问题不是“没人做”，而是目标拆解、进度同步和结果整合成本太高。

| 维度 | 真实情况 |
|------|----------|
| 触发点 | 新产品发布、客户投诉升级、合规整改、季度经营分析等跨部门任务 |
| 现有材料 | 总目标、截止时间、已有文档、相关制度、待决策事项和参与角色 |
| 卡点 | 子任务多、依赖关系不清、负责人分散，管理者需要不停催进度和整合结果 |
| DesireCore 介入 | 核心调度拆解任务、分派智能体、跟踪阻塞项，并把需要人决策的事项集中呈现 |
| 验收结果 | 管理者看到统一进度、交付物和待拍板清单，而不是在多个群里找信息 |

## 涉及智能体

- **DesireCore 核心** -- 任务编排、智能体调度、全局监控
- **法律顾问** -- 合规审查
- **AI 文书** -- 营销材料撰写
- **财务助手** -- 成本核算和定价
- **产品经理** -- 发布时间表和任务分配

## 完整对话示例

### 一键启动多部门协作

```
你: 我们要在下个月发布新产品，需要准备发布相关的所有材料。

DesireCore: 我来帮你协调这次产品发布。我将它拆解为以下工作：

        1. 法律合规审查 -- 检查产品文案、用户协议
        2. 营销材料准备 -- 产品介绍、宣传文案、社媒内容
        3. 财务成本核算 -- 发布成本、定价策略
        4. 项目规划 -- 发布时间表、任务分配

        已委派各智能体开始工作：
        → 法律顾问：审查产品合规性
        → AI 文书：准备营销材料框架
        → 财务助手：进行成本预估
        → 产品经理：制定发布计划

        预计 30 分钟内返回初步方案。
```

### 实时进度监控

```
你: 现在进展怎么样了？

DesireCore: 各项工作进展如下：

        法律合规审查：已完成
        - 产品文案无合规问题
        - 用户协议需补充数据处理条款

        营销材料：进行中（70%）
        - 产品介绍初稿已完成
        - 社媒内容矩阵制作中

        财务核算：已完成
        - 发布预算：85 万
        - 建议定价区间：299-399 元

        发布计划：已完成
        - 关键里程碑已标注
        - 资源分配无冲突

        需要你确认的事项：
        1. 用户协议数据处理条款（法律顾问已草拟）
        2. 定价选择 299 还是 399（财务助手已给出分析）
```

### 紧急任务插队

```
你: 紧急！重要客户投诉了产品质量问题，需要立即处理。

DesireCore: 收到紧急任务，正在调整资源优先级...

        当前状态：
        - AI 文书：正在制作社媒内容（可暂停）
        - 法律顾问：空闲

        调度决策：
        → 暂停 AI 文书的社媒任务（已保存进度）
        → 委派客服中心全力处理投诉
        → 法律顾问准备可能的应对方案

        预计 5 分钟内首次回复客户。
```

## 核心能力

### 任务编排

DesireCore 能将一个复杂需求拆解为多个子任务，识别任务间的依赖关系，安排串行或并行执行：

- 无依赖的任务并行执行，缩短总耗时
- 有前后依赖的任务按顺序串联
- 自动监控每个任务的完成状态
- 把每个子任务绑定交付物、负责人和截止时间

### 智能调度

当系统资源紧张或出现意外情况时，DesireCore 会：

- 根据任务优先级重新分配资源
- 紧急任务自动抢占低优先级任务的资源
- 某个智能体故障时，自动将任务转派给备用智能体
- 将阻塞项集中展示给你，而不是散在多个对话里

### 意图识别

DesireCore 能区分你是在**教学（Teach）**还是**委派任务（Delegate）**：

- "以后写邮件用轻松语气" → 教学，记录偏好并同步给所有文案类智能体
- "帮我写封邮件给客户" → 委派，分配给 AI 文书执行

### 决策点管理

跨部门任务最容易卡在“没人拍板”。DesireCore 会把需要你决定的事项单独列出来：

| 决策点 | 来源 | 影响 | 下一步 |
|--------|------|------|--------|
| 定价选 299 还是 399 | 财务助手 | 影响毛利和市场定位 | 你确认定价后，AI 文书更新宣传材料 |
| 用户协议数据条款是否接受 | 法律顾问 | 影响上线合规 | 你确认后同步产品经理排上线 |
| 发布日期是否延期 | 产品经理 | 影响营销节奏和客服准备 | 你确认后重新调整任务依赖 |

## 最终成果

一个可跟踪的多智能体协作流程：任务被拆解、分派、汇总和复核，最终形成统一的交付物、进度表、阻塞项和待决策清单。

## 会用到的 DesireCore 能力

- **智能任务编排**：识别意图、拆解子任务、分配智能体、跟踪进度并汇总回执
- **团队模式 / 蜂群模式**：长期角色负责稳定流程，临时角色处理一次性复杂任务
- **决策树与人闸门**：关键分支和高风险动作需要你确认后再继续
- **回放与审计**：复盘每个智能体做了什么、哪个环节耗时、哪里需要优化

:::tip 最佳实践
- 描述需求时说清全局目标（如"发布新产品"）、截止时间、现有材料和不可变约束
- 对于有明确截止时间的项目，一开始就告知时间约束，调度器会优先保障关键路径上的任务
- 定期检查进度汇总，及时对需要你决策的事项做出回应，避免阻塞其他智能体的后续工作
:::
