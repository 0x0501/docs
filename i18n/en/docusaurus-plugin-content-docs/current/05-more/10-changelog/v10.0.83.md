# DesireCore v10.0.83

**Release Date**: 2026-07-12

![v10.0.83 更新概览](/img/changelog/v10.0.83-en.png)

## New Features
- **Claude Subscription Integration**: Support for logging in via Claude Code OAuth or manual configuration, using Claude Pro/Max subscription compute; model calls are fulfilled through the built-in official SDK backend with complete audit trails.
- **Thinking Depth Control**: Added thinking depth level buttons in the chat input area, supporting Off / Auto / Deep multiple levels, with per-session memory; covers OpenAI, Anthropic, and domestic model dialects.
- **Three-Tier Prompt System**: Added global (personal) and team-level prompts, injected on top of Agent-level prompts; new centralized Prompt Center for unified management, editing, and preview of all three tiers.
- **Migrate to New Device Wizard**: A new “Migrate to New Device” entry in Settings, with a five-step wizard to export all data; on the new machine, the import process automatically recognizes the migration package, supports identity comparison, pre-backup protection, and restart to apply.
- **Merge into Current Identity**: When importing a migration package, supports “Merge into Current Identity” mode, which remaps source identity data to the local identity name while keeping the local identity unchanged.
- **Encrypted API Key Migration**: Optionally carry custom Provider API keys during migration, encrypted with a passphrase and stored independently (original secrets files are never included in the package); upon import, decide merging for each Provider individually.
- **File “Open With” Dropdown**: File entries in chat messages now have an “Open With” menu, supporting in-app open, system default app, choose another app, and locate in file manager.
- **Windows GUI Automation (CUA Driver)**: The Windows installer ships with cua-driver, enabling Agents to control desktop applications (screenshot, click, type, read window). Background operations do not steal focus, and users get out-of-the-box experience with zero setup.
- **Transparent Context Overrun Self-Healing**: When the context exceeds the model’s true limit, the system automatically learns the real window and transparently retries; for large outputs, dynamic budget calculation produces the result in one go, eliminating manual adjustments.
- **Video Generation Upgrade**: Upgraded to OpenAI Video API standard format, integrated official cloud compute for image/video generation, and adapted for the Doubao Seedance 2.0 video model.
- **Compute Preset Update Notification**: When official Provider or model configurations published by config-center change, a notification badge and review panel show conflict diff and change details; users can adopt or keep each item.
- **Built-in Runtime Offline Installation**: The installer bundles Python 3.13 and Node 24 LTS offline runtimes; they are silently imported in the background on first launch, enabling environment setup without internet.
- **Structured Prompt Editing**: Agent persona and principles now support L0/L1/L2 layered form editing, with automatic format and schema validation on save; the system prompt preview panel shows segment proportions and allows copying the full prompt.
- **Manual Refresh on Apps & Services Page**: Added a refresh button to manually synchronize registry directories and the pending approval queue.
- **Stealth Task Board Probing**: Switching sessions no longer floods the console with 404 error logs from probing non-existent task boards.
- **Relationship Graph Now Shows Delivery Edges**: After an Agent successfully delivers results, the “delivery” relationship edge is correctly displayed in the relationship graph, forming a closed loop with the “delegation” edge.
- **Compute Page Shows Preset Configuration Version**: The header metadata area displays the current preset data version, aiding in troubleshooting configuration sync issues.
- **Queued Queries Can Be Cancelled**: Users can stop a request at any time while waiting in the queue, preventing unconditional scheduling and execution.
- **Top-Level Query Concurrency Limit Configurable**: The number of simultaneous top-level queries can be customized via an environment variable; interactive queries automatically take priority over background tasks.

## Fixes
- **Cursor Not Following Line Breaks in Chat Input**: After operations like Shift+Enter line break, accepting prediction suggestions, or history recall, the cursor now scrolls into the visible area automatically.
- **Data Corruption in Concurrent Multi-Agent Delegation**: Fixed data consistency issues during concurrent delegation, including team directory conflicts, member loss, half-written file corruption, and task count drift.
- **Scheduled Tasks Not Executed**: Fixed an issue where scheduled tasks would reply with a single confirmation and then be truncated without ever completing; failed tasks now correctly back off and alert instead of infinitely retrying.
- **Horizontal Scrollbar in Message Area**: Long URLs and wide Markdown tables no longer force a horizontal scrollbar; content is always fully visible.
- **Crash When Clicking file:// Links**: Clicking a local file link in an Agent’s reply no longer crashes or grays out the window; instead it is opened with the system default app.
- **Two Types of 400 Errors from DeepSeek-v4 Could Not Self-Heal**: Gateway `max_tokens` range limit errors and missing thinking block return errors are now automatically learned, corrected, and transparently retried.
- **Scan with Non-Vision Model Degenerates into Repetition**: When reading a pure image-scan PDF using a non-vision model, the system now prompts to switch models directly instead of clearing tools, which previously caused the model to endlessly repeat output.
- **Chat List Indicator Misleading**: The list indicator light no longer incorrectly shows green while an Agent is executing tools; it now correctly displays yellow (busy).
- **Market Pointer Skill Invisible**: ADR-038 pointer-type skills are once again visible in the marketplace, searchable, and viewable in detail; version update detection works correctly.
- **Missing ProviderId for Default Model**: Fixed a problem where historical data lost the ProviderId for the default model configuration, causing the default to become permanently invalid.
- **credentialSource Poison Pill**: Fixed an issue where older clients merging a Provider preset containing unknown credential sources would reject the entire configuration, causing all compute to fail; now only unsupported Providers are skipped, and the rest work normally.
- **Global Concurrency Slot Leak from Aborted Queries**: Repeatedly stopping queries manually no longer exhausts global concurrency slots, which previously froze all Agents. Also fixed leak paths triggered by system sleep and session reset.
- **Request Misrouting for Identically Named Models**: When multiple Providers have identically named models, requests are no longer incorrectly routed to an unrelated Provider.
- **Imported Agent List Offline**: Imported or cloned Agents now appear online in the list immediately without requiring an application restart.
- **Missing Validation for Fan-out Delegation Targets**: When delegating to multiple targets simultaneously, invalid targets are now explicitly reported as failed instead of being silently skipped; the delegation card correctly shows all target names.
- **AgentFS Hot Reload Failure**: Changes made by directly editing files like `agent.json` now take effect instantly without relying on manual refresh.
- **Mismatched Async Acknowledgment for send_message**: Concurrent async message acknowledgments are no longer incorrectly assigned to other waiting synchronous requests.
- **Mismatched Reply for Concurrent via-messaging Requests**: When the same caller makes multiple delegation/message/assist requests to the same target concurrently, replies are now precisely matched to the corresponding request.
- **Window Flicker When Scrolling Past Bottom of Chat**: Continued scrolling after reaching the bottom of the message list no longer triggers window flickering.
- **Chat Message Action Bar Overflow**: Action buttons beneath short messages no longer overflow and cover the timestamp.
- **Unresponsive Selection Circle in Multi-Select Mode**: Fixed an issue where clicking the selection circle was unresponsive in certain scenarios.
- **Market Repository Source Saving Stuck**: After adding a new repository source, the interface no longer gets stuck on “Saving” or shows repeated progress.
- **Relationship Heat Value Overflow**: Relationship heat on the agent list is displayed as thousands with one decimal place, no longer bursting the card.
- **Cannot Cancel During Pipeline Assembly Phase**: The tool assembly and context preparation phase (after acquiring a slot but before the actual query) now responds to stop operations.
- **Service Directory Rendering Crash**: Dirty data generated when an Agent registers a service on behalf of someone no longer crashes the pending approval page; directory status updates in real time.
- **Duplicate Team Creation with Same Name**: Duplicate team creation by AI repeated calls is now automatically deduplicated, preventing redundant teams.
- **Orphaned Teams After Deleting Agent**: Teams that contained the deleted agent are automatically disbanded or have the member removed, preventing orphaned teams.
- **Scanned Document Tool Clearance Causing Degenerative Repetition with Non-Vision Model**: Now directly prompts to switch model instead of letting the model fall into a dead loop with no available tools.
- **Provider Concurrent Queue Without Timeout or Cancel**: Queued requests that are disconnected or aborted are immediately dequeued and no longer blocked.
- **Failed to Clear Data on Windows**: Fixed errors when clearing data caused by read-only git files and reserved device name files.
- **Git Operations Failed on Windows**: Fixed failures in market sync, Agent snapshots, etc., caused by git binary paths containing spaces or colons.
- **Missing Cancel Checkpoint During Pipeline Assembly Phase**: Long-running steps like MCP tool discovery now respond to cancellation.
- **Missing OpenAI Entry Point for Claude SDK Backend**: OpenAI-format requests are now correctly routed to the Claude SDK backend.

## Improvements
- **Context Trimming and Token Estimation Optimization**: Trimming now uses a cheapest-first cascading strategy, with lossless unloading of re-fetchable tool results; token estimation now covers previously missed thinking and tool names, and non-OpenAI models automatically include additional margin.
- **Provider Concurrency Limiting**: Each Provider can have a maximum concurrent request limit and queue size; excess requests wait in a queue instead of immediately failing. Configuration takes effect instantly.
- **Wider Chat Scrollbar Hit Area**: The message list scrollbar track width increased to 10px for easier dragging.
- **Relaxed Timeout for Synchronous Sub-Task Waiting**: Default wait time for delegation, `spawn_agent`, `send_message`, etc., extended from 5 minutes to 30 minutes; upon timeout, the task is no longer aborted automatically but instead prompts the caller to decide.
- **Ls Tool Default No Approval**: Directory listing now executes directly; on macOS, first access to a protected directory triggers a system authorization dialog, and a permission denied response includes guidance.
- **SSRF Protection Down to Per-Agent Control**: Each Agent can enable internal network protection individually, and DNS rebinding attacks are also blocked.
- **Message Multi-Select Circle Accessibility Improved**: Selection circles are now semantic buttons, supporting keyboard operation and screen readers.
- **Relaxed WorkflowRun Wait Timeout**: Running workflows is no longer truncated by a fixed timeout; a custom timeout can be passed.
- **Declarative Marking of Re-fetchable Tools**: The “re-fetchable” attribute of a tool is now declared in the registry, and MCP tools automatically derive it based on the protocol.
- **Memory Message Range Fix**: Message counts in thread details no longer show abnormally large values (billions); historical dirty data is auto-repaired.
- **Cloud Models Display Partitioned by Type**: Official compute models are displayed in partitions (text / image generation / video generation), with pricing shown by billing mode and supplier announcements at the top.
- **System Prompt Windows Path Protection**: Windows paths in system prompts are wrapped in inline code to prevent backslash escaping by Markdown.
- **Persona/Principles Read/Write Performance Optimization**: Uses caching based on file modification time, re-parsing only after external edits.
- **Agent Capability to Configure Compute**: Agents can help users configure Providers and API keys; keys are write-only for Agents.
- **Enhanced Migration Compatibility**: Old-version export packages and future-version packages can be safely imported, with proactive alerts when importing across identities.
- **Delegation Prompt Adds Path Rules**: When passing file resources across Agents, absolute paths are automatically used.
- **Compute Key Security Hardening**: Keys are automatically masked in HTTP responses; plaintext viewing requires IPC channel authorization. Read/Grep/Ls tools are precisely denied access to key files.
