# DesireCore v10.0.49

**发布日期**: 2026-05-08

![v10.0.49 更新概览](/img/changelog/v10.0.49.png)

## 新功能
- **DashScope 编码方案支持增强**：新增 DashScope Token 方案并完善相关模型与提供商预设，让你在配置编码与算力方案时有更多可直接使用的 DashScope 选项。

## 修复
- 智能体执行“检测更新”时，如果远程仓库暂时无法访问或网络异常，现在会明确提示获取远程版本失败，而不会再误报“已是最新版本”。
- 智能体自动更新遇到 Git 冲突时，现在会直接显示冲突文件，并在聊天头部持续显示冲突标记，方便你定位并处理问题。
- 新建智能体后，Git 仓库初始化更稳定，紧接着进行 Git 初始化、查看状态或历史时不再容易因并发冲突而失败。
- 智能体自动更新时会优先按仓库配置的分支拉取，减少因分支不一致导致的更新异常。
- 自动更新过程中，像未配置远程仓库、配置无效、当前目录不是 Git 仓库等不同失败原因，现在会分别给出更准确的提示，而不是统一显示通用失败信息。

## 改进
- 手动检查桌面应用更新时，现在会按你当前操作系统拉取对应平台的版本列表，因此能更早看到属于自己平台的新版本。
- 主动检查智能体更新时会绕过旧缓存并重新获取远端信息，刚发布的新版本能更及时被检测到。
