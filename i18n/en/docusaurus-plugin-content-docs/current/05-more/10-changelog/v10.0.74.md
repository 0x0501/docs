# DesireCore v10.0.74

**Release Date**: 2026-06-06

![v10.0.74 更新概览](/img/changelog/v10.0.74-en.png)

## New Features
- **PDF Document Outline Reading**: Agents can now return a PDF’s chapter outline and page numbers first, helping you locate the sections you need more quickly in long documents.
- **Export Documents as PDF**: Agents now have built-in support for exporting documents directly as PDF, without relying on temporary scripts or additional conversion steps.

## Fixes
- Fixed an issue where Agent PDF document export was not working; PDF files can now be generated properly through the relevant export APIs.
- Fixed formatting issues when exporting PDFs from the editor. Exported tables, code blocks, heading anchors, and page content are now more complete and better structured.
- Fixed issues where reading PDFs with very large pages or extremely long content could cause continuation errors or freezing, making long-document reading more stable.

## Improvements
- Optimized the speed and stability of agents when reading long PDF documents. Re-reading the same document is now faster, reducing wait time caused by repeated parsing.
- Improved the PDF reading strategy by prioritizing text extraction and automatically stopping ineffective parsing once the context budget is reached, reducing lag and unnecessary resource use.
- Optimized reading performance for complex PDF pages such as chart pages and scanned pages. Even when the current model is not a vision model, it can still return as much extractable text content as possible.
- Improved handling of extremely long PDFs. When too many pages are requested, the system no longer rejects the request outright, but instead reads the first 50 pages first and prompts you to continue reading.
- Optimized PDF page reading that requires vision delegation. Recognition results for complex pages are now more complete, and repeated reads are also faster.
- Unified the rendering styles for editor exports and Agent-generated PDFs, making overall export results more consistent.
